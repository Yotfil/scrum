¿Qué es scrum?

Es un marco de trabajo (Framework) el cual se centra en la idea desarrollo ágil, pero ¿Qué es el desarrollo ágil?. Es la capacidad de adaptarse y manejar cambios, con el objetivo de desarrollar software de forma eficiente. El objetivo de manejar scrum en un proyecto es el de crear productos escalables, en un tiempo que está definido por varios desarrollos y entregas, teniendo en cuenta que la satisfacción del cliente es prioritario, por este motivo SCRUM maneja entregas parciales. Otro de los objetivos de SCRUM es el trabajo colaborativo, ya que esta metodología no basa en una pirámide de cargos, sino que todos las personas del equipo tiene voz y voto y todas tiene un rol importante para el cumplimiento con satisfacción del producto.

Herramientas

La metodología scrum tiene unas herramientas muy importantes las cuales están en todo el ciclo de vida del proyecto.

  • Backlog

    Elementos que conforman el proyecto y es una descripción general del proyecto, entre los elementos se pueden tener en cuenta bugs. Los backlogs son son requerimientos oficiales del proyecto.

  • Historias de usuario

    Permite dar información acerca del funcionamiento del requerimiento, generalmente las historias de usuario son los requerimientos del proyecto.

  • Sprint

    Son los requerimientos del proyecto, normalmente, cada una de las historias de usuario se convierten en sprint a desarrollar.

  • Tablero de tareas

    Es donde se mostrará el listado de tareas que se tiene

Roles

Es un marco Son las responsabilidades que va a tener cada integrante del equipo:

  • Product Owner

    Es la persona que tiene la visión de lo que el cliente quiere y el encargado de transmitir esta visión al equipo, debido a que es la “cara del cliente”, es el usuario principal de sistema.

    El perfil de un Product Owner debe ser una persona con conocimientos de usuarios, debe saber leer e interpretar las necesidades del cliente, pero no solo se centra en los clientes, también debe ser un gran motivador del equipo.

  • Scrum Master

    Es un mentor, el cual se encarga de acompañar al equipo durante todo el proceso de desarrollo asegurándose de mantener las buenas prácticas y siendo intermediario al momento de resolver problemas.

    El perfil de un Scrum Master es una persona con habilidad de comunicación, motivación y negociación.

  • Lider de desarrollo

    Es la persona encargada de coordinar el trabajo del equipo de desarrollo, pero no solo es coordinar, esta persona también debe ser un mentor para su equipo.

    El perfil de un Líder de Desarrollo es una persona con amplios conocimientos en las herramientas utilizadas ya que si se presentan inconvenientes, el líder debe tener la capacidad de guiar a la solución del mismo.

  • Equipo de desarrollo

    Son las personas encargadas de escribir el código, realizar pruebas para identificar errores o mejoras.

    Suelen existir dos equipos de desarrollo, unas personas las cuales estarán encargadas del diseño, maquetación y manejo de colores. Otras personas las cuales se encargaran de la lógica del proyecto.

Srint

Son cada una de las iteraciones que se tendrán en un proyecto scrum, esto quiere decir que son las actividades que se tendrán en el desarrollo de un proyecto. La división del proyecto en sprints permite tener entregas parciales, en las cuales se puede tener una visión de cómo va el proyecto, las mejoras que se deben tener y nos asegura tener un mejor control en momentos de cambios. La duración de cada sprint debe estar entre 2 semanas a máximo dos meses.

Los sprint tienen unos eventos los cuales tiene el objetivo de supervisar y analizar el estado del sprint, ellos son:

  1. Planeación: Durante este evento, todo los miembros del equipo se reúne para analizar los requerimientos del sprint y definir tareas y tiempos, donde a cada integrante se le asigna una tarea por hacer y cada persona identifica el tiempo que le tomará realizar esta tarea. Una vez terminada la planificación del tiempo, se dará un tiempo a sprint.

  2. Reuniones: Durante cada sprint, el equipo se reunirá para hablar sobre el estado de cada tarea. Durante esta reunión se hablará sobre lo que se hizo el día anterior, que se hará el día de hoy y qué obstáculos se encontraron. El objetivo de estas reuniones es saber como va el equipo y buscar el apoyo entre todos ellos, si existe algún inconveniente, el equipo debe buscar una solución colaborativa.

  3. Revisión del sprint: El Product Owner, Scrum Master y el Cliente se reúnen para mostrar el trabajo desarrollado, se demuestra que todos los requerimientos solicitados se cumplieron y se analiza si existe alguna modificación o corrección a realizar.

  4. Retrospectiva: Una vez terminada la reunión con el cliente, todo el equipo se reúne para hablar sobre lo trabajado en el sprint y se analizan temas como que se puede mejorar, que se hizo bien y qué obstáculos se presentaron.
